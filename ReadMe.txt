Le but de ce projet était de développer une interface graphique d’utilisateur qui permettra la configuration des routeurs dans un réseau.
Le principal avantage de développer une telle application était d’éviter les configurations via ligne de
commande, en offrant une interface plus intuitive à l’utilisateur, qui, en théorie, n’aurait plus besoin de
connaître les commandes spécifiques des configurations.

Rôle des fichiers : 
	- View.py gère l'affichage de l'IHM
	- Model.py gère la gestion et manipulation des données de l'application
	- Controller.py gère le reste de l'application
	- database.csv est la base de données par défaut, si l'utilisateur commence un projet à zéro
	- style.css gère le design de l'application

Rôle des dossiers : 
	- Logo contient tous les icônes de l'application
